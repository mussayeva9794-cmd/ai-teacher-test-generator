# Demo Script

## 2-3 Minute Defense Flow

1. Open the app and show the clean `Create` screen.
   - Explain that the teacher sets topic, grade level, test type, language, pedagogical goal, and subject tags.

2. Generate a test.
   - Mention that the app builds four variants automatically:
     - `Variant A` - easy
     - `Variant B` - medium
     - `Variant C` - hard
     - `Variant D` - mixed

3. Open `Review`.
   - Show one question in the editor.
   - Explain that the teacher can edit wording, options, correct answer, explanation, and skill tag.
   - Point to the `Quality Report` and mention duplicate detection, explanation checks, and mixed-variant balance checks.

4. Open `Share`.
   - Create a student link for one variant.
   - Mention attempt limits, deadlines, and separate Student/Teacher exports.

5. Open the shared student page.
   - Show the progress bar, cleaner mobile-friendly layout, draft saving, and submission confirmation.

6. Submit one attempt.
   - Explain that the response is saved immediately and appears on the teacher page within seconds.

7. Open `Analyze`.
   - Show:
     - student cards
     - class average
     - weak topics
     - variant comparison
     - student journal
     - per-student progress across multiple tests

8. Finish with the project value.
   - This is not just a generator.
   - It supports the full teacher workflow:
     - create
     - review
     - share
     - collect responses
     - analyze results
